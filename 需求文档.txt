# 无人机点位素材上传管理系统需求文档
## 一、项目概述
### 1.1 项目背景
现有福州沿海共140个测绘点位，多人使用无人机拍摄图片、视频素材，素材统一在电脑端整理后上传自有服务器。为解决素材与点位匹配混乱、人工归集效率低问题，开发轻量化PC端素材上传网页工具，仅临时项目使用，项目结束可一键销毁服务与全部数据。

### 1.2 项目目标
1. 仅PC端网页，完全不做移动端适配；
2. 双页面权限隔离：作业上传页（无密码公开访问）、管理员后台页（独立密码校验）；
3. 图片前端画质压缩控体积，视频不做任何压缩，仅做大小拦截；
4. Docker容器一键部署，内置SQLite嵌入式文件数据库，无需独立数据库服务；
5. 公网IP+端口纯HTTP访问，无需域名；
6. 管理员页面支持单独下载素材，附带实时下载进度条；
7. 超限视频弹出清晰轻量化压缩指引，降低用户操作成本；
8. 仅图片支持在线预览，视频关闭在线播放功能，节约服务器公网带宽。

### 1.3 适用范围
- 使用人员：无人机拍摄作业人员（仅上传素材）、项目管理员（查看/下载/删除素材）
- 运行环境：自有Linux服务器，Docker容器化部署，公网IP+端口HTTP访问
- 终端限制：仅电脑端使用，手机、平板设备不做兼容适配

## 二、整体架构与部署要求
### 2.1 部署方案
1. 全容器化打包：前端静态页面、后端服务、SQLite数据库统一容器管理，提供`docker-compose.yml`一键启停；
2. 数据库：SQLite嵌入式文件数据库，无需额外启动数据库服务，数据目录挂载宿主机持久化；
3. 网络访问：公网IP+自定义端口访问，无域名、无HTTPS，纯HTTP协议；
4. 持久化规则：素材文件、数据库文件、分片临时缓存全部映射宿主机目录，删除容器不会丢失业务数据；
5. 资源轻量化：适配低配服务器，支持十余人同时并行上传操作。

### 2.2 宿主机挂载目录结构
```
./upload-web
├── data
│   ├── db.sqlite       # SQLite数据库文件
│   ├── temp_chunk      # 分片临时文件目录，服务自动定时清理7天过期分片
│   └── storage         # 所有图片、视频素材存储根目录
└── docker-compose.yml  # 容器部署配置文件
```

## 三、页面功能需求（仅PC端，无移动端适配）
### 3.1 页面1：作业人员上传页面（无密码公开访问）
#### 3.1.1 点位选择与全局状态可视化
1. 内置完整140条福州点位固定数据，下拉选择框展示格式：`序号 | 区县 | 岸段类型 | 经纬度`；点位数据固化，前后端均不提供编辑、新增、删除点位能力；
2. 顶部全局点位状态点阵：展示全部140个点位序号圆点，**图片+视频全部上传完成显示绿色圆点，未全部上传显示红色圆点**，鼠标悬浮可弹出该点位完整信息；
3. 强制操作流程：必须先选中点位，上传按钮才会解锁，未选择点位时上传按钮置灰禁用；
4. 同点位重复上传弹窗确认：若当前点位已有图片/视频，再次上传对应类型文件时弹出确认框，询问是否覆盖原有素材。

#### 3.1.2 文件上传、压缩与大小限制规则
1. 文件格式白名单：仅允许 `jpg/png/webp` 图片、`mp4` 视频，其余格式直接拦截禁止上传；
2. 图片处理规则：
   - 不限制原始分辨率，仅压缩画质控制体积；
   - 单张原图上限20MB，超出直接弹窗拒绝上传；
   - 前端压缩至20MB以内后生成Blob再发起上传，原始大图不直接传输；
3. 视频处理规则：
   - **全程不做任何前端压缩、转码、分辨率调整**；
   - 前端预先校验文件体积，原始视频文件≥100MB直接拦截，弹出指引弹窗；
   - 弹窗提示文案：`当前视频超过100MB限制，请将视频拖拽至微信文件传输助手/好友发送，微信会自动压缩视频，保存压缩后的视频再上传`；
4. 分片上传逻辑：单文件大于10MB自动切分分片传输，支持断点续传，网络中断后无需重新选择文件，可直接续传；
5. 进度展示：分别展示图片压缩进度、分片上传进度条；上传成功/失败弹出提示，失败提供一键重试按钮；
6. 权限约束：作业人员仅可提交素材，无素材下载、删除、批量查看全部点位素材功能。

### 3.2 页面2：管理员后台页面（密码校验进入）
#### 3.2.1 登录鉴权
1. 打开页面先弹出密码输入框，管理员密码写在后端.env配置文件内，支持自行修改；
2. 仅单管理员固定密码，无用户注册、多账号功能；
3. 所有管理类接口携带临时Token鉴权，上传页面无法绕过密码调用管理接口，无Token返回403禁止访问。

#### 3.2.2 点位总览列表
1. 完整展示140条点位信息，字段包含：点位序号、区县、岸段类型、图片上传状态、视频上传状态、最后上传时间；
2. 支持按素材完成状态筛选（全部/仅传图/仅传视频/全部完成）。

#### 3.2.3 素材管理与下载能力
1. 素材预览规则：**仅图片支持缩略图在线预览，视频不提供在线播放窗口，不加载视频流节省公网带宽**；
2. 单文件下载：图片、视频分开独立下载，下载过程页面展示实时下载进度条；
3. 文件删除：可单独删除点位图片、单独删除点位视频，或一键清空当前点位全部素材。

## 四、后端接口通用需求
1. 分片上传接口：接收分片数据、校验分片完整性、全部分片上传完毕后自动合并完整文件；
2. 定时清理任务：自动清除7天前未合并的过期分片临时文件，避免磁盘冗余；
3. 后端二次校验：再次校验文件后缀、文件体积，拦截损坏、超限、非法文件；
4. 状态同步机制：每次上传覆盖、删除素材后，实时更新SQLite数据库内对应点位素材状态；
5. 接口权限划分
   - 公开接口（上传页免鉴权）：点位列表查询、分片上传提交、断点续传校验；
   - 私有管理接口（需管理员Token）：点位详情查询、素材下载、文件删除。

## 五、SQLite嵌入式数据库设计
### 5.1 点位基础表 point_info
项目初始化自动导入140条固定点位数据，运行期间不可修改
| 字段 | 说明 |
|------|------|
| id | 点位序号（主键） |
| city | 地市 |
| district | 区县 |
| lon | 客户需求经度 |
| lat | 客户需求纬度 |
| shore_type | 岸段类型 |

### 5.2 点位素材记录表 point_material
记录每个点位图片、视频上传状态与存储路径
| 字段 | 说明 |
|------|------|
| id | 自增主键 |
| point_id | 关联点位序号，外键绑定point_info.id |
| img_path | 图片服务器存储路径，为空代表未上传图片 |
| video_path | 视频服务器存储路径，为空代表未上传视频 |
| upload_time | 最后一次素材上传时间戳 |

## 六、Docker部署交付要求
1. 交付完整可运行 `docker-compose.yml`，执行 `docker-compose up -d` 一键启动整套服务；
2. 配套 `.env` 环境配置文件，可自定义公网访问端口、管理员登录密码、分片大小、单文件上限；
3. 所有业务数据目录挂载宿主机，容器销毁素材、数据库不丢失；
4. 附带极简部署操作文档：服务启停步骤、管理员密码修改方法、素材本地存放路径说明、项目结束清理数据操作。

## 七、非功能性需求
1. 并发性能：支持10人同时并行上传，分片上传无阻塞卡顿；图片前端压缩不会造成浏览器卡死；
2. 公网HTTP安全控制：
   - 管理员专属接口Token鉴权，外部人员无法下载、删除素材；
   - 严格校验上传文件后缀，拦截exe、压缩包等危险文件；
3. 带宽优化：关闭视频在线播放功能，仅图片缩略图预览，大幅降低公网流量消耗；
4. 可维护性：代码注释清晰，部署文档简单易懂；项目结束直接删除宿主机挂载目录即可清理全部数据。

## 八、明确不开发功能（排除项）
1. 不开发移动端页面、手机拍照录像上传功能；
2. 不支持域名、不做HTTPS加密；
3. 无多账号、注册登录体系，仅单管理员固定密码；
4. 不使用MySQL、PostgreSQL等独立数据库，仅使用SQLite嵌入式文件库；
5. 不对接第三方云存储，素材仅本地存储在自有服务器；
6. 视频无任何前端/后端转码、压缩逻辑；
7. 无短信、邮件、消息推送通知功能；
8. 无在线图片、视频编辑功能，仅支持上传、预览、下载、删除；
9. 不提供批量打包下载、点位Excel导出功能；
10. 不做素材自动按点位分文件夹归类逻辑；
11. 不提供视频在线播放功能。
